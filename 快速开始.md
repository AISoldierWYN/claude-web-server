# 快速开始

## 从 Git 克隆后（首次部署）

仓库**不包含**本机配置文件（避免把密钥与路径提交上去）。克隆到本地后，请在项目根目录**各复制一份模板**再按需修改：

**Windows（PowerShell 或 CMD）**

```powershell
copy config.example.ini config.ini
copy claude_web_paths.config.example.json claude_web_paths.config.json
```

**macOS / Linux**

```bash
cp config.example.ini config.ini
cp claude_web_paths.config.example.json claude_web_paths.config.json
```

- **`config.ini`**：服务端口、鉴权、Claude CLI 等（见主 README「配置文件详解」）。
- **`claude_web_paths.config.json`**：技能包 `bundles`、只读目录等；不需要技能包时可保留示例中的空结构或最小 JSON。

## 最小准备

1. **Python 3.10+**（建议）与 **pip**（Linux 上常用 `python3` / `pip3`）。
2. 本机已安装 **Claude Code CLI**（`claude` 在 PATH，或于 `config.ini` → `[claude]` → `cli_path` 写绝对路径），并已按官方方式登录/配置 API。
3. 项目根目录已存在 **`config.ini`**（见上一节从 `config.example.ini` 复制）。

**说明**：服务端支持 **Windows 与 Linux**（及 macOS），无需单独开关；可选在 `config.ini` → `[features]` → `v3_linux_deploy = true` 仅在日志中标记「Linux 部署」场景（详见主 README）。

## 必装依赖

```bash
pip install -r requirements.txt
```

## 最少需要改的配置

| 项 | 说明 |
|----|------|
| 一般 **不用改** | 默认监听 `0.0.0.0:8080`，无 Token，数据在仓库下 `cache/`、`logs/`。 |
| 需要鉴权时 | 在 `config.ini` 的 `[auth]` 里填 `token = 你的密钥`，或启动时：`python server.py 你的密钥`。 |
| 指定 Claude 可执行文件 | 若 `claude` 不在 PATH，在 `config.ini` → `[claude]` → `cli_path` 写完整路径。 |
| 技能包 / 只读目录（可选） | 编辑 `claude_web_paths.config.json`，见主 README。 |

其余参数（端口、模型、权限模式、上传大小等）均可先使用 `config.ini` 中的默认值。

## 启动

```bash
python server.py
```

（Linux 上若 `python` 未指向 3.x，请使用 `python3 server.py`。）

浏览器访问：`http://127.0.0.1:8080`（端口以 `config.ini` 中 `[server] port` 为准）。

## 防火墙（局域网其它设备访问）

端口须与 **`config.ini` → `[server]` → `port`** 一致（默认 **8080**）。

**Windows**（管理员 PowerShell）：

```powershell
netsh advfirewall firewall add rule name="Claude Web Server" dir=in action=allow protocol=tcp localport=8080
```

**Linux**（示例，`ufw`）：

```bash
sudo ufw allow 8080/tcp
sudo ufw reload
```

更完整的参数说明见 **`README.md`** 中的「配置文件详解」章节。
